## ðŸ§© 4.1 Defining a Function

```rust

    println!("Hello from Rust!");
}

fn main() {
    greet();
}
```

- `fn` defines a function
- `greet()` is called from `main`

ðŸ§  Functions **must be declared before use**, or you'll get a compile error (unless declared later with the right visibility).

## ðŸ§© 4.2 Function Parameters

```rust
fn greet(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    greet("Alice");
}
```

- `name: &str` means a borrowed string slice.
- In Rust, we usually pass strings by reference unless we **want ownership**.

## ðŸ§© 4.3 Returning Values

```rust
fn square(num: i32) -> i32 {
    num * num
}
```

- The arrow `-> i32` is the **return type**
- Rust returns the value of the **last expression** (no `return` needed)

âœ… This works:

```rust

    return num * num; // with return
}
```

âœ… And this is more idiomatic:

```rust

    num * num // no semicolon â†’ expression returns the value
}
```

> â— A semicolon at the end turns it into a **statement**, which returns `()` (unit type).

## ðŸ§© 4.4 Multiple Parameters


## ðŸ§© 4.5 Expression vs Statement (again)

Remember:

```rust

    let x = 5;
    x + 1 // no semicolon â†’ returns 6

```

If you did:

```rust

    let x = 5;
    x + 1; // semicolon â†’ returns `()`, not i32 â†’ compile error!
```
